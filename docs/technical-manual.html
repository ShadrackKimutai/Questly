Questly Technical Guide

A developer and operator reference for Questly, the self-hosted real-time quiz application built with React, Vite, Socket.IO, and TypeScript.

Version 0.2.5 pnpm monorepo React 19 Socket.IO 4 JSON file storage

Overview

Questly lets a host create and run quiz games while players join with an invite code. The host uses the manager interface, players use the party interface, and both communicate with a Socket.IO server over the /ws path.

Manager workflow Authenticate, select or edit a quiz, create a room, control rounds, view results, and review saved games.
Player workflow Enter room code, choose a name and mascot, answer questions, view feedback, and track the leaderboard.
Persistence Quiz definitions, game config, and completed results are stored as JSON files in the configured config/ directory.

Architecture

Browser |-- React SPA served by Vite/nginx |-- Socket.IO client connects to /ws | v Socket server on port 3001 |-- handlers: game, manager, quiz, results |-- services: registry, config, game engine |-- in-memory active games | v config/ |-- game.json |-- quiz/*.json |-- results/*.json

The repository is split into three workspace packages. Shared types, constants, and validators live in packages/common. The game engine and file persistence live in packages/socket. The user interface lives in packages/web.

Setup

Prerequisites

  • Node.js 22 or newer, matching the project README.
  • pnpm 10.16 or newer.
  • Docker and Docker Compose if using the container workflow.

Install and run locally

pnpm install
pnpm run dev

pnpm run dev starts the web and socket packages in parallel. The web dev server runs on port 3000 and proxies /ws to the socket server on port 3001.

Production build

pnpm run build
pnpm start

Configuration

The socket service initializes the config directory through initConfig(). Set CONFIG_PATH to control where files are written. 

config/
  game.json
  quiz/
    example.json
  results/
    game-result-id.json
File Purpose
game.json Stores managerPassword, the password required to access the manager interface.
quiz/*.json Stores editable quiz definitions.
results/*.json Stores completed game results for later review.

Socket Package

The socket package starts a Socket.IO server on port 3001 with the path /ws. It registers handlers for game, manager, quiz, and results events.

Core services

  • services/registry.ts: tracks active games and invite codes.
  • services/game/index.ts: owns a single game room, manager state, player state, reconnect logic, and game actions.
  • services/game/round-manager.ts: controls countdowns, question flow, answer collection, scoring, responses, leaderboard, and saved result history.
  • services/config.ts: reads and writes config, quiz, and result JSON files.

Common Package

The common package is the contract between the server and the web app. Change it carefully because both sides compile against these types. 

export type QuestionType =
  | "single"
  | "multiple"
  | "truefalse"
  | "shortanswer"
  | "wordcloud"

Shared constants include EVENTS, NO_TIME_LIMIT, MEDIA_TYPES, MASCOTS, and EXAMPLE_QUIZ.

Game Flow

  1. The manager authenticates with managerPassword.
  2. The manager selects a quiz and emits game:create.
  3. The server creates a Game, assigns a UUID and invite code.
  4. Players join by invite code and choose a username.
  5. The manager starts the game.
  6. Each round moves through prepare, question, answer, result, responses, and leaderboard states.
  7. When the last question is complete, the server saves a result JSON file.
Active games are in memory. Completed games are persisted under config/results.

Question Types

Type Player input Correctness Review behavior
single One option index. Correct when selected index is in solutions. Shows correct or incorrect.
multiple Array of option indices. Correct when selected set exactly matches solutions. Shows correct or incorrect.
truefalse One of two option indices. Same as single choice. Shows correct or incorrect.
shortanswer Free text. Correct when text matches one of textSolutions, case-insensitive and trimmed. Shows correct or incorrect.
wordcloud Free text. Ungraded participation response. Should use a neutral state such as "Response" or "Submitted", not correct or incorrect.

Example quiz entry

{
  "question": "Enter your favourite color in a rainbow",
  "type": "wordcloud",
  "answers": [],
  "solutions": [],
  "cooldown": 5,
  "time": 20,
  "media": {
    "type": "image",

Results Review

Completed games are saved as GameResult records. Each question stores its original question fields plus playerAnswers. 

interface PlayerAnswerRecord {
  playerName: string
  answerId: number | number[] | string | null
}

Correctness states

State When to use Suggested label
Correct Choice or short-answer response matches the configured solution. Correct
Incorrect Choice or short-answer response does not match the configured solution. Incorrect
Neutral Wordcloud response, because it is intentionally ungraded. Response or Submitted
No answer The player did not submit anything before the timer ended. No answer
Wordcloud responses should not reduce correctness percentages or be rendered as incorrect in the review table. They represent submitted open-ended responses rather than graded answers.

Socket Events

Socket event names are centralized in packages/common/src/constants.ts. Event payload types are defined in packages/common/src/types/game/socket.ts.

Namespace Direction Events
GAME server to client game:status, game:reset, game:updateQuestion, game:playerAnswer
PLAYER client to server player:join, player:login, player:selectedAnswer, player:selectedAnswers, player:textAnswer, player:changeMascot
MANAGER client to server manager:auth, manager:startGame, manager:nextQuestion, manager:showLeaderboard, manager:kickPlayer
QUIZ Both quiz:get, quiz:data, quiz:save, quiz:update, quiz:delete
RESULTS Both results:get, results:data, results:delete

Status States

The server drives the UI through game:status. Each status maps to a player or manager component in features/game/utils/constants.ts.

SHOW_ROOM SHOW_START SHOW_PREPARED SHOW_QUESTION SELECT_ANSWER SHOW_RESULT SHOW_RESPONSES SHOW_LEADERBOARD FINISHED WAIT

Troubleshooting

Players cannot connect

  • Confirm the socket server is running on port 3001.
  • Confirm nginx or Vite proxies /ws to port 3001.
  • Check browser console and socket server logs for connection errors.

Manager cannot log in

  • Check config/game.json.
  • Use the managerPassword field.
  • Confirm CONFIG_PATH points to the expected config directory.

Quiz does not appear

  • Ensure the file is in config/quiz.
  • Ensure it is valid JSON.
  • Ensure it passes the Zod quiz validator.

Wordcloud responses appear incorrect

Treat wordcloud as neutral in result review. A submitted wordcloud answer is a response, not a graded answer. The review UI should branch on questionResult.type === "wordcloud" and show a third state.
